# План міграції на React

Гілка: `react-migration`. Мета — перевести застосунок на React так, щоб:
- **код виглядав солідно** (типізовані компоненти, єдиний роут-конфіг, без HTML-рядків у TS);
- **нічого не зламалось** (формули/розрахунки лишаються байт-у-байт тими самими модулями);
- **сторінки були налаштовані під СММ** (унікальні title/description на вкладку, Open Graph / Twitter-картки).

## Чому React (а не Angular)

Проєкт уже на Vite + TypeScript — React під'єднується одним плагіном без зміни збірки.
Angular вимагає власного CLI/збірки, RxJS і повного переписування одразу; React дозволяє
**інкрементальну** міграцію (strangler pattern) з робочим застосунком після кожного кроку.

## Роутинг: нормальні шляхи без хеша

Вкладки живуть на справжніх URL: `/refine`, `/doll`, `/genie`, … (history API).
Старі хеш-посилання (`/#doll`) редиректяться на шляхи разово при завантаженні.

Чому без react-router: роутинг тривіальний (18 плоских вкладок), власний хук
`useRoute()` (~50 рядків) робить усе потрібне — pushState/popstate, редирект
legacy-хешів, робота з підпапки GH Pages (`/pw.calc/doll`) через
`import.meta.env.BASE_URL` — без зайвої залежності.

Інфраструктура для history-роутингу на GitHub Pages:
- база збірки в CI — абсолютна (`VITE_BASE=/pw.calc/`), інакше ассети ламаються
  на вкладених шляхах;
- `scripts/prerender.mjs` (пост-білд) генерує **`dist/<route>/index.html` на кожну
  вкладку з унікальними `<title>`/description/og:*/canonical** — тому прямі лінки
  віддаються зі статусом 200, а краулери соцмереж (без JS!) бачать правильні
  прев'ю кожної сторінки — це і є «файне СММ»;
- `dist/404.html` — SPA-фолбек для невідомих шляхів;
- `sitemap.xml` + `robots.txt` (коли задано `SITE_URL`).

## Фази

### Фаза 1 — React-оболонка (цей коміт) ✅
- `src/app/routes.meta.json` + `src/app/routes.ts` — єдине джерело правди про
  вкладки: id, група, підпис, іконка, **title, description** (його ж читає
  пререндер при білді).
- `src/app/useRoute.ts` — history-роутинг (див. вище).
- `src/components/` — `Header`, `Sidebar`, `SubtabsBar`, `Layout`, `Footer`,
  `GlobalModals` (ідіоматичний JSX; сайдбар/сабтаби генеруються з роут-конфіга).
- `src/pages/*.tsx` — 18 сторінок, розмітка сконвертована з `index.html` в JSX.
  Сторінки статичні й **завжди змонтовані** (як старі `<section class="tab-panel">`),
  активна перемикається класом — це зберігає поведінку legacy-модулів, які тримають
  слухачі на елементах усередині; `React.memo` гарантує, що React не чіпає DOM,
  який мутують модулі.
- `src/app/legacyInit.ts` — ініціалізація всіх калькуляторів (колишній `main.ts`)
  викликається один раз після монтування React-дерева. **Модулі розрахунків не
  змінювались узагалі** — вони працюють із тим самим DOM за тими самими id.
- `index.html` → мінімальний: head (FOUC-скрипти, шрифти, OG-меta) + `#root`.
- Без `StrictMode` до фази 3: подвійне dev-монтування перетворює DOM панелей,
  а legacy-слухачі вішаються один раз.

### Фаза 2 — СММ/меta (цей коміт) ✅
- `<PageMeta>`: на зміну вкладки оновлює `document.title`, description, og:*.
- Пререндер per-route HTML з унікальними OG-тегами + canonical + og:url,
  `sitemap.xml`, `robots.txt` (див. розділ про роутинг вище).

### Фаза 3 — поступова конверсія сторінок в ідіоматичний React (roadmap)
Порядок від простих до складних, після кожної — видалення відповідного legacy-коду.
Модуль shards/data.ts лишається спільним джерелом даних (рецепти, дропи).
1. `guides` (статика) → чистий JSX без init.
2. `settings` (форми) → контрольовані компоненти + контекст налаштувань.
3. `chests`, `eggs`, `compare`, `craft` (прості калькулятори) → стан у хуках,
   формули виносяться в чисті функції `src/lib/` і покриваються юніт-тестами.
4. `r8` — вже статична React-таблиця (без legacy).
5. ✅ `r8sim`, `gsn`, `simulator`, `refine`, `skills`/`genie`/`pets` —
   конвертовано (обчислення винесені в src/lib/*; simulator має рушій-двигун
   через useSyncExternalStore; genie-калькулятор — src/lib/genieCalc.ts).
6. ✅ `doll` — сконвертовано. Формуло-критична логіка винесена byte-у-байт у
   чисті `src/lib/doll/*` (stats/buffs/damage/tooltip/summary/types); рушій
   `engine.ts` + шар даних `data.ts` лишились без змін. Сторінка —
   `src/pages/DollPage.tsx` + модалки `src/pages/doll/DollModals.tsx`
   (стан білду в localStorage, useMemo-похідні, 4 модалки, drag-drop/ПКМ/тач,
   тултіпи через спільний util). Еквівалентність доведена headless-звіркою з
   legacy-еталоном (зведення/тултіпи/дамаг — 0 розбіжностей) + інтерактивні
   смоуки. Видалено legacy `src/modules/doll/index.ts` і `dollInit`.
7. ⏸ Свідомо лишений як legacy-панель: `rb` — інтерактивна карта на Leaflet
   (третьосторонній імперативний API: тайли, маркери, fullscreen). Обгортати
   в React-ідіоми — це переписування заради нульового виграшу.

### Фаза 4 — SEO/СММ поглиблено (виконано в межах фаз 1–2)
History-роутинг, prerender per-route HTML, 404-фолбек, sitemap/robots — уже
реалізовані (див. розділ про роутинг). Залишок фази 4: OG-зображення на
вкладку (замість спільного favicon-180).

### Старий текст фази 4 (для історії)
- Перехід на history-роутинг (`/doll` замість `#doll`) + `404.html`-фолбек для
  GH Pages, редиректи зі старих хеш-URL.
- Prerender вкладок при білді (vite-plugin-prerender / власний скрипт через
  headless) → повні OG-прев'ю для кожної сторінки.
- `sitemap.xml` + `robots.txt`.

## Інваріанти (перевіряються після кожної фази)
- Всі 18 вкладок рендеряться без JS-помилок (headless-смоук).
- Формули: зведення ляльки, заточка, тултіпи — значення збігаються з main.
- Старі посилання `#<tab>` працюють; back/forward працюють.
- `vite build` проходить; деплой лишається «push у main» (гілка не деплоїться).
